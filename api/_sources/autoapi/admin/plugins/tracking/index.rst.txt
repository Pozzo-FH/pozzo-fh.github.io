.. py:class:: IdleTracker(min_players_for_kick=MIN_PLAYERS_FOR_KICK, balance_afks_before_minplayers=BALANCE_AFKS_BEFORE_MINPLAYERS, max_idle_time=MAX_IDLE_TIME, check_interval=IDLE_CHECK_INTERVAL, yaw_threshold=IDLE_YAW_THRESHOLD)



   Kick/balance players who are AFK

   Player idle time is reset when:
   a) spawns (event)
   b) connects (event)
   c) chat message (event)
   d) kills/gets killed (event)
   e) moves -> yaw changes (check)

   Important edge cases:
   1) commander in commander chair, doing stuff like supply/bomb (keep)
   1b) commander in commander chair, doing nothing (kick)
   2) arty, turning yaw  (keep)
   2b) arty with fixed yaw that scores kills (keep)
   2c) arty not killing, fixed yaw
   3) idle when dead (remove)
   4) idle in vehicle/on foot (remove)

   Depending on number of players, either kick AFKs, spread them, or nothing

   Disable for admins, and entirely on Beta

   POSSIBLE EXTENSIONS to keep in mind:
   - Possible to disregard the timer reset if you die in the
     same location that you spawned?
