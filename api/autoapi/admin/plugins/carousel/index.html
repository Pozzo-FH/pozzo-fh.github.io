class admin.plugins.carousel.Carousel¶

Map rotation, picking random maps with filtering

New maps are rolled during the start of each round, so the next map can be overridden by the !setnext command.

The layer (size) of a map is chosen at the end of a map, since during a round, alot of players might (dis-)connect

random_map(self)¶

Pick a random nextmap

Map start: - roll a map - reroll if duplicate theatre - reroll if not supporting current playernumber - pick layer based on current player num

Map end: - pick new layer based on endgame playernum - if not possible, keep previous

Exceptions are for Favorite maps and map themes (ignoring the no-double- theatre rule)

idx, mapname, layer, gamemode
